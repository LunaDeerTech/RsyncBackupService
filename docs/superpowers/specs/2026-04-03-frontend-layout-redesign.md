# Rsync Backup Service — 前端页面布局与信息架构重设计

> 日期: 2026-04-03
> 状态: 待确认

---

## 1. 概述

本文定义 Rsync Backup Service 前端页面的全局骨架、信息架构、页面布局模式和交互规则的完整重设计方案。

### 1.1 重设计动机

当前前端页面存在以下核心问题：

1. **表单与列表同屏**：多个页面（实例列表、策略、存储目标、SSH 密钥、通知渠道）采用左右双栏布局，表单和列表挤在同一屏幕，编辑时列表不可见，移动端体验差。
2. **Settings 页面过度拥挤**：用户管理、密码修改、权限矩阵、会话信息全部堆在一个页面的 4 张卡片中。
3. **视觉层级不够**：页面内各区域权重相似，缺乏主次分明的信息层级，管理、概览、操作、风险提示混在同一层。
4. **导航结构扁平**：侧边栏将所有管理功能平铺，缺乏分组，管理员和普通用户看到的层级不清晰。

### 1.2 设计目标

- 建立清晰的信息架构，侧边栏精简为 5 项以内
- 统一所有 CRUD 页面为「全宽列表 + Modal 表单」交互模式
- 区分管理员与普通用户的导航和功能范围
- 强化实例级概览（mini Dashboard），补充运行时数据与丰富 KPI
- 将低频管理功能合并为「系统管理」多 Tab 页面

### 1.3 非目标

- 不重写自定义组件库（AppButton、AppTable 等）——只调整页面布局和使用方式
- 不修改后端 API——前端重设计基于现有接口
- 不改变设计语言（Balanced Flux 风格、token 体系、色彩系统保持不变）

## 2. 信息架构

### 2.1 选定方案：合并精简型

将低频管理页面（SSH 密钥、通知渠道、审计日志、用户管理）合并为「系统管理」多 Tab 页面，侧边栏精简为 5 项。

### 2.2 路由结构

**公共路由（无需认证）：**

| 路径 | 页面 | 说明 |
|------|------|------|
| `/login` | LoginView | 登录页 |

| 路径 | 页面 | 说明 |
|------|------|------|
| `/` | DashboardView | 全局仪表盘 |

| `/storage-targets` | StorageTargetsView | 存储目标管理 |

| 路径 | 页面 | 说明 |

普通用户登录后默认跳转 `/instances`（实例列表），不展示全局仪表盘。管理员默认跳转 `/`（仪表盘）。

## 4. 页面布局模式

### 4.1 核心模式：全宽列表 + Modal 表单

所有包含 CRUD 操作的页面统一采用以下模式：

- **列表区域**占据全部内容宽度，不与表单共享空间
- **新建/编辑操作**通过居中 Modal（带遮罩层）进行
- **Modal 宽度**：480px ~ 560px，响应式适配

### 4.2 Modal 使用场景

| 场景 | 触发方式 | Modal 类型 |
|------|----------|-----------|
| 创建/编辑资源 | 页面顶部按钮或表格行操作 | 表单 Modal（较宽） |
| 危险确认（删除、恢复、重置密码） | 操作按钮 | 确认 Dialog（较窄，danger tone） |
| 连通性测试（SSH、存储目标） | 表格行操作 | 操作 Modal |
| 发送测试通知 | 行内操作 | 操作 Modal |
| 分配实例权限 | 用户列表行操作 | 表单 Modal（实例勾选列表） |

### 4.3 受影响页面清单

| 页面 | 当前布局 | 新布局 |
|------|----------|--------|
| 实例列表 | 左列表+右表单同屏 | 全宽列表 + Modal 新建/编辑 |
| 实例详情-策略 Tab | 左列表+右密集表单同屏 | 全宽列表 + Modal 新建/编辑策略 |
| 实例详情-恢复 Tab | 左表单+右记录同屏 | 全宽记录表格 + Modal 发起恢复 |
| 实例详情-通知订阅 Tab | 左列表+右表单同屏 | 全宽列表 + Modal 添加/编辑订阅 |
| 存储目标 | 左分组表格+右表单同屏 | 全宽分组列表 + Modal 新建/编辑 |
| SSH 密钥 | 独立页左列表+右表单 | 系统管理 Tab 内列表 + Modal 登记 |
| 通知渠道 | 独立页左列表+右表单+测试 | 系统管理 Tab 内列表 + Modal 编辑 |
| Settings | 4 卡片全堆一页 | 拆分为系统管理 4 Tab + 个人信息独立页 |

## 5. 各页面设计规格

### 5.1 全局仪表盘（DashboardView）

保持当前布局方向，主要变化是占据全部内容宽度。

**布局结构：**

```
页面标题 + 刷新按钮
├── KPI 卡片行（4 列）
│   ├── 实例总数
│   ├── 今日备份
│   ├── 成功率 / 成功数
│   └── 失败任务
├── 双栏区域
│   ├── 运行中任务（实时 WebSocket 更新）
│   └── 系统状态
└── 双栏区域
    ├── 最近备份时间线
    └── 存储空间概览表格
```

**与当前的差异：** 无结构性变化，仅因全宽布局获得更好的呼吸空间。

### 5.2 实例列表（InstancesListView）

**布局结构：**

```
页面标题 + [+ 新建实例] 按钮
├── 筛选栏（搜索 + 启用状态筛选）
├── 中继模式警告通知（条件显示）
└── 实例表格（全宽）
    ├── 列：名称、数据源、状态、上次备份、操作
    └── 操作：编辑（→ Modal）、详情（→ 路由跳转）
```

**「新建实例」Modal 字段：** 名称、数据源类型、数据源路径、主机/端口/用户/SSH 密钥（remote 时）、排除规则、启用开关。

### 5.3 实例详情（InstanceDetailView）

**Tab 结构（保持 5 Tab）：**

| Tab | 管理员 | 普通用户 | 内容 |
|-----|--------|---------|------|
| 概览 | ✓ | ✓ | mini Dashboard |
| 策略 | ✓ | ✗ | 策略列表 + Modal 编辑 |
| 备份历史 | ✓ | ✓ | 筛选 + 表格 |
| 恢复 | ✓ | ✗ | 记录表格 + Modal 发起恢复 |
| 通知订阅 | ✓ | ✗ | 订阅列表 + Modal 编辑 |

#### 5.3.1 概览 Tab（V3-C：KPI 卡片行 + 双栏详情）

**布局结构：**

```
实例名称 + [启用] 标签
├── KPI 卡片行（5 列）
│   ├── 活跃策略数
│   ├── 备份总数
│   ├── 累计容量
│   ├── 成功率
│   └── 最近备份状态+时间
├── 双栏区域
│   ├── 源配置（类型、路径、排除规则等定义列表）
│   └── 当前运行任务（实时进度条，无任务时显示空态）
└── 最近活动（备份时间线：时间、状态、策略名→目标、容量）
```

**与当前的差异：**

- 增加 KPI 卡片行（策略数、备份总数、累计容量、成功率、最近状态）
- 移除创建者信息（仅管理员可创建实例，无实际意义）
- 增加当前运行任务的实时进度展示
- 增加最近备份活动时间线（含失败详情）

#### 5.3.2 策略 Tab

**布局结构：**

```
[+ 新建策略] 按钮
├── 策略表格（全宽）
│   ├── 列：名称、备份类型、调度方式、保留策略、启用状态、操作
│   └── 操作：编辑（→ Modal）、删除（→ 确认 Dialog）
```

**策略表单 Modal 字段：** 名称、备份类型、启用开关、调度模式（定时/间隔）、Cron/间隔值、最大执行时间、保留天数、保留数量、冷备卷大小（cold 类型时）、存储目标多选。

#### 5.3.3 备份历史 Tab

保持当前布局，已经合理：筛选器（备份类型、状态、策略）+ 全宽表格 + 刷新按钮。无结构性变化。

#### 5.3.4 恢复 Tab

**布局结构：**

```
中继模式警告（条件显示）
├── 恢复记录表格（全宽）
│   ├── 标题行含 [发起恢复] 按钮
│   ├── 列：恢复源、目标路径、状态、覆盖模式、时间
│   └── 空态提示
```

**「发起恢复」Modal 流程：**

1. 选择快照/归档源（下拉选择）
2. 填写恢复目标路径
3. 覆盖模式开关 + 动态风险警告通知（danger tone）
4. 确认区域：输入当前密码验证身份
5. 提交按钮（danger 样式）

#### 5.3.5 通知订阅 Tab

**布局结构：**

```
[+ 添加订阅] 按钮
├── 订阅表格（全宽）
│   ├── 列：渠道名称、事件类型、收件人、启用状态、操作
│   └── 操作：编辑（→ Modal）、删除（→ 确认 Dialog）
```

**订阅 Modal 字段：** 渠道选择、收件人邮箱、事件类型多选（backup_success、backup_failed、restore_complete、restore_failed）、启用开关。

### 5.4 存储目标（StorageTargetsView）

**布局结构：**

```
页面标题 + [+ 新建目标] 按钮
├── 滚动备份目标（分组标题）
│   └── 表格（名称、类型、路径、用量、更新时间、操作）
└── 冷备份目标（分组标题）
    └── 表格（名称、类型、路径、用量、更新时间、操作）
```

**用量列：** 每行显示一个多色横条图，展示该存储目标的磁盘使用状况：

- 备份占用（品牌色 primary）
- 其他文件占用（中性灰）
- 剩余空间（低对比度填充）
- 悬停时 tooltip 显示具体数值（备份 XX GB / 其他 XX GB / 剩余 XX GB / 总计 XX GB）

**注意：** 用量数据依赖 API 返回的存储目标统计。若当前 API 不返回用量信息，此功能标记为 v1.1 增强，v1 版本仍以当前 `available_bytes` + `backup_count` 显示。

**操作列：** 编辑（→ Modal）、测试连通性（→ 操作 Modal）、删除（→ 确认 Dialog）。

### 5.5 系统管理（SystemAdminView）

**仅管理员可见。** 包含 4 个 Tab：

#### Tab 1: 用户管理

```
[+ 创建用户] 按钮
├── 用户表格（全宽）
│   ├── 列：用户名、角色标签、创建时间、操作
│   └── 操作：分配实例（→ Modal）、重置密码（→ 确认 Dialog）
```

**「分配实例」Modal：** 以用户为中心的权限分配流程：

- 标题显示目标用户名
- 列出所有备份实例，每行一个勾选框
- 勾选的实例为该用户授予 viewer 权限
- 可选：每个实例旁的下拉选择角色（viewer / admin）
- 保存按钮提交权限变更

**「创建用户」Modal 字段：** 用户名、密码、是否管理员开关。

**「重置密码」Dialog：** 输入新密码 + 当前管理员密码确认，danger tone 样式。

#### Tab 2: SSH 密钥

```
[+ 登记密钥] 按钮
├── 密钥表格（全宽）
│   ├── 列：名称、指纹、创建时间、操作
│   └── 操作：测试连通性（→ Modal）、删除（→ 确认 Dialog）
```

**「登记密钥」Modal 字段：** 密钥名称、私钥文件路径。

**「测试连通性」Modal 字段：** 选择密钥、目标主机、端口、用户名、测试按钮。

#### Tab 3: 通知渠道

```
[+ 新建渠道] 按钮
├── SMTP 渠道表格（全宽）
│   ├── 列：名称、类型、启用状态、更新时间、操作
│   └── 操作：编辑（→ Modal）、测试（→ Modal）、删除（→ 确认 Dialog）
```

**「新建/编辑渠道」Modal 字段：** 名称、SMTP 主机、端口、发件地址、用户名、密码（可选，空值保留现有）、TLS 开关、启用开关。

**「测试通知」Modal：** 输入测试收件人邮箱、发送测试按钮。

#### Tab 4: 审计日志

```
├── 筛选栏（操作类型、资源类型、用户、开始时间、结束时间）
├── 应用/重置按钮
└── 日志表格（全宽，分页）
    ├── 列：时间、用户、操作、资源类型、详情（JSON）
    └── 分页器（上一页/下一页）
```

**与当前的差异：** 移除右侧时间线卡片（原只显示前 8 条，与表格内容重复），改为全宽表格布局。

### 5.6 个人信息（ProfileView）

## 7. 交互规则

### 7.1 Modal 行为

- 打开时显示半透明遮罩层
- ESC 键或点击遮罩层关闭（表单有未保存数据时需确认）
- Modal 内表单提交成功后自动关闭并刷新列表
- 加载中状态禁用提交按钮，显示 Spinner

### 7.2 危险操作流程

所有危险操作（删除资源、恢复数据、重置密码）遵循统一流程：

1. 触发按钮使用 danger tone 样式
2. 弹出确认 Dialog，展示操作后果
3. 需要输入当前密码验证身份（恢复、重置密码时）
4. 确认按钮同样使用 danger tone

### 7.3 空态处理

列表/表格为空时统一使用 `AppEmpty` 组件，显示操作引导文案：

- 无实例时：「还没有备份实例。点击上方按钮创建第一个。」
- 无策略时：「该实例尚未配置备份策略。」
- 无存储目标时：「请先创建存储目标以启用备份。」

### 7.4 权限控制

- 路由级别：`requiresAdmin` meta 字段控制管理员专属路由
- 组件级别：侧边栏导航按角色过滤显示
- Tab 级别：实例详情页 Tab 按角色动态渲染
- 按钮级别：普通用户不显示创建/编辑/删除等操作按钮
