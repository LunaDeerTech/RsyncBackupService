# 登录页重设计规格

> 日期: 2026-04-04
> 状态: 已确认方向，待进入实现规划

## 1. 背景

当前登录页虽然已经使用了项目的前端组件与主题 token，但整体结构仍带有明显的阶段性展示页特征：大面积 Hero、能力清单、较强的“欢迎区”表达、默认预填 `admin`。这会削弱它作为正式产品登录入口的可信感。

本次改版目标不是增加更多视觉表现，而是把登录页收敛成真正面向产品使用的入口页面。

## 2. 设计目标

- 让登录页看起来像正式产品入口，而不是 UI 风格 demo 页面
- 强化登录动作本身的优先级，弱化展示性内容
- 保留项目既有的 `Balanced Flux` 品牌基调，但控制表达强度
- 不修改登录接口、鉴权流程、路由跳转和错误处理语义
- 同时覆盖桌面端和移动端的稳定显示

## 3. 非目标

- 不重写登录流程或会话逻辑
- 不新增登录页之外的业务功能
- 不做双栏欢迎页、能力介绍区或营销式首页结构
- 不在登录页继续保留开发演示痕迹

## 4. 选定方案

本次登录页采用“居中单卡片 + 顶部品牌标题”的结构。

选择原因：

- 比双栏欢迎区更接近正式产品入口
- 比极简裸卡片多一层识别度，但不会重新滑回展示页气质
- 更适合与现有前端 token、卡片、输入和按钮组件衔接

## 5. 页面结构

### 5.1 页面外层

- 页面背景使用冷静、低对比的浅色基底
- 允许使用极弱的品牌色氛围，不再使用大 Hero 渐变主导版面
- 页面中央垂直与水平居中放置登录区域
- 登录卡片上方仅保留产品标题和一句简短说明

### 5.2 登录卡片

卡片只保留以下内容：

- 登录标题
- 简短说明文案
- 用户名输入框
- 密码输入框
- 错误提示
- 登录按钮
- 次级位置的主题切换入口
- 登录后跳转说明

以下内容移除：

- 能力概览列表
- 展示型欢迎区
- 阶段性建设说明
- 默认预填 `admin`

## 6. 视觉规则

- 卡片采用明确边界、克制阴影和稳定实体感
- 品牌色只用于按钮、焦点环、弱背景氛围和局部强调
- 标题与辅助文案使用正式、产品化表达，不再使用 demo 导向的语气
- 主题切换从主视觉元素降为次要辅助操作
- 错误提示保留在表单区域内，保证反馈与动作邻近

## 7. 文案方向

文案应偏“正式控制台入口”而非“欢迎页”表达。

建议方向：

- 顶部标题：`Rsync Backup Service`
- 顶部说明：一句简短的控制台入口说明
- 卡片标题：`登录到控制台`
- 卡片说明：强调使用账户访问受保护页面
- 底部提示：保留登录后返回原目标页面的说明

## 8. 响应式规则

- 桌面端保持单卡片居中，卡片宽度收敛到适合快速登录的范围
- 移动端延续同一结构，仅压缩外边距和卡片内边距
- 不为移动端引入不同的页面叙事结构

## 9. 实现边界

- 主要修改 `web/src/views/LoginView.vue`
- 优先复用现有 `AppCard`、`AppButton`、`AppFormField`、`AppInput`、`AppPasswordInput`、`AppNotification`
- 不重做或绕开已有 UI 基础组件
- 允许调整测试以匹配新的结构与文案，但登录行为断言必须保留

## 10. 验证要求

- 登录页提交流程测试继续通过
- 前端构建通过
- 本地浏览器可打开登录页进行视觉预览
- 实现完成后询问用户效果是否符合预期
