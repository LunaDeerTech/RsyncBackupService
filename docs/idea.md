# RsyncBackupService

我计划开发一个基于 Rsync 的备份服务，提供以下核心功能：

- 支持滚动备份、冷备份；
- 滚动备份为增量备份，冷备份为全量备份；
- 数据源和备份目标可以是本地磁盘、远程服务器、云存储等；
- 数据源支持本地、ssh远程，备份目标支持本地、ssh远程、云存储（仅冷备份）；
- 计划备份可以设置定时间隔或cron表达式；
- 一个备份实例可以配置多个计划备份；
- 可以手动触发计划备份；
- 冷备份可以选择支持压缩和加密；
- 具备审计日志功能，记录服务的操作和备份历史；
- 用户分为管理员admin和只读用户viewer，管理员可以管理备份策略和用户权限，viewer只能查看特定示例的备份历史和日志；
- 冷备份可以下载到本地，增量备份不提供下载功能；
- 支持备份数据的恢复功能，可以选择恢复到原位置或指定位置；
- 提供Web界面进行管理和监控；
- 可以设置备份保留策略，自动删除过期的备份数据；

## 技术选型

- 滚动备份：基于 rsync 工具实现增量备份（--link-dest）；
- 后端：使用 go 语言；
- 数据存储：使用 SQLite 存储备份元数据和日志；
- 前端：使用 Vue 3 + Vite 开发单页应用，支持桌面、移动端，支持浅色和深色主题；
- 前端风格：使用 Tailwind CSS 开发自定义的组件样式库，设计风格参考文档 [Rsync Backup Service — 前端组件样式设计](./component-style-design.md)；
- 认证授权：使用 JWT 实现用户认证和权限控制；
- 部署：将前端打包进后端，提供单一可执行文件部署；

## 功能模块

### 仪表盘

- 仅管理员可见；普通用户登录后不显示仪表盘，默认进入其有权限的实例列表；
- 页面目标：
    - 快速判断系统当前是否健康；
    - 快速发现待处理风险；
    - 快速进入实例、策略、备份目标、审计日志等页面处理问题；
- 顶部总览卡片：
    - 运行中任务数；
    - 异常实例数；
    - 待处理风险数；
    - 系统容灾率（综合评分）-风险等级-趋势；
    - 备份目标健康度（可达性、已用容量、剩余容量）；
- 运行态与风险：
    - 当前任务与队列：任务名称、实例名称、类型（滚动/冷备份/恢复）、当前步骤、进度、开始时间、预计完成时间；
    - 风险提醒：长时间未备份、连续失败、冷备份缺失、备份目标不可达、备份目标容量不足；
- 趋势与分布：
    - 实例健康分布（健康/警告/严重）；
    - 最近备份结果趋势（24小时/7天）；
    - 即将执行的计划任务日历；
- 重点关注实例：
    - 列出当前最需要关注的 5~8 个实例；
    - 展示实例名称、当前状态、最近一次备份结果-时间、容灾率、风险原因摘要；
    - 点击进入实例详情；
- 快捷入口：
    - 实例列表；
    - 备份目标；
    - 远程配置；
    - 审计日志；
- 仪表盘不承载完整管理操作，只负责总览、预警和跳转；


### 实例列表

- 管理员可见所有实例；普通用户仅可见其有权限的实例；

- 创建实例：打开弹窗
    - 输入实例名称
    - 选择数据源类型（本地/ssh远程/云存储）：
        - 本地：输入本地路径
        - ssh远程：输入远程服务器地址、用户名、选择配置的[密钥](#ssh密钥管理)
- 实例列表：名称、数据源、当前状态（备份中/空闲）、上次备份结果（成功/失败）-时间、查看详情

#### 实例详情

- 概览：实例名称、数据源、当前状态、上次备份结果-时间、[容灾率](#容灾率计算)：
    - 当前任务进度（如果有任务正在进行）：
        - 任务名称、类型（滚动/冷备份/恢复）
        - 显示任务进度条
        - 当前步骤（如正在备份/正在压缩/正在加密等）
        - 开始时间、预计完成时间、剩余时间
    - 备份总数（条形图）：当前实例的备份总数/系统总备份数
    - 备份总容量（条形图）：当前实例的备份总容量/系统总备份容量
    - 成功率（计算百分比）：成功备份数/总备份数
    - 备份历史列表（5条最近的备份记录）：类型、结果、大小、持续时间、时间
    - 计划任务列表（5条最近的计划任务）：名称、类型、启动时间·倒计时；

- 策略列表：
    - 列表内容：名称、类型（滚动/冷备份）、目标、上次执行结果-时间、累计成功率、操作（编辑/触发/删除）
    - 新增/编辑策略：
        - 基本信息：名称、类型（滚动/冷备份）
        - 目标：根据类型选择目标配置项（见 [备份目标](#备份目标)）
        - 计划：选择计划类型（定时间隔/cron表达式），输入相应的计划参数
        - 冷备份选项：是否启用压缩、是否启用加密（勾选加密需要额外配置加密密钥）、是否分卷（如果启用分卷，需要输入分卷大小）
        - 保留策略：选择保留策略类型（按时间/按数量），输入相应的参数（保留天数/保留数量）

- 备份列表：
    - 列表内容：完成时间、类型（滚动/冷备份）、备份大小、实际大小、开始时间、持续时间、操作（恢复/下载（下载仅冷备份））
    - 恢复功能：
        - 选择恢复类型（源位置/指定位置）
            - 如果选择指定位置，输入目标路径
            - 要求输入实例名称、当前用户密码（验证用户权限、确认操作）
            - 点击确认后触发恢复操作，回到实例详情页面，显示恢复任务进度

- 审计日志：
    - 列表内容：时间、操作类型（创建实例/编辑实例/删除实例/创建策略/编辑策略/删除策略/手动触发备份/备份结果）、操作人、详情
    - 支持按时间范围和操作类型筛选日志记录

- 实例设置：
    - 基础信息：实例名称、数据源类型（本地/ssh远程）
    - 访问权限：选择用户、配置用户权限（管理员/只读/只读+下载备份）


### 备份目标

- 仅管理员可见；普通用户登录后不显示备份目标菜单；

- 滚动备份目标列表：名称、备份类型（滚动/冷备份）、存储类型（本地/ssh远程/云存储）、备份容量-已用容量-剩余容量（条形图）、操作（编辑/删除）
- 新增滚动备份目标：
    - 输入名称、选择备份类型（滚动/冷备份）
    - 存储类型：
        - 滚动备份可选择：本地/ssh远程
        - 冷备份可选择：本地/ssh远程/云存储
    - 输入存储路径（本地路径/远程服务器地址/云存储路径）
    - 操作（编辑/删除）
- 备份目标健康检查：
    - 周期性检测目标可达性、已用容量、剩余容量、最近成功检查时间、最近失败原因；
    - 本地目标检查磁盘可写性与容量，ssh远程与云存储检查连通性与容量；
    - 检查结果用于目标信息展示、仪表盘展示与[容灾率计算](#容灾率计算)；

### 系统配置

- 仅管理员可见；普通用户登录后不显示系统配置菜单；

#### 远程配置

- 远程存储列表：名称、类型（ssh远程/云存储）、容量（已用-总容量）、创建时间、操作（编辑/删除）
- 新增远程存储：
    - 基础信息：输入名称、选择类型（ssh远程/云存储）
    - ssh远程：输入远程服务器地址、用户名、上传私钥文件、输入私钥密码（如果有）、测试连接（点击测试连接按钮，显示连接结果）

## 补充说明

### 页面布局框架

- 整体采用左右布局，左侧为导航栏，右侧为内容区；

- 导航栏包含：仪表盘、实例列表、备份目标、系统配置、个人中心，点击后在内容区展示对应页面；
- 在桌面端：导航栏固定在左侧，不随页面滚动；内容区右侧显示当前页面内容；
- 在移动端：导航栏隐藏在顶部，点击菜单按钮展开；内容区占满整个屏幕显示当前页面内容；

- 内容区顶部为标题、介绍；
- 采用 tab 分页展示不同功能模块，如实例详情中的概览、策略列表、备份列表、审计日志等；
- 采用表格展示列表数据，支持分页、排序、筛选等功能；
- 采用 modal 弹窗展示新增/编辑表单，表单内容根据不同功能模块进行调整；

- 通知类信息（如操作结果、风险事件等）采用顶部或底部的 toast 形式展示，颜色和图标根据信息类型（成功/失败/警告）进行区分；

### 配置文件

- 采用 .env 存储基础配置：
    - `DATA_DIR`：数据目录，数据库、缓存数据、日志等；
    - `PORT`：服务监听端口；
    - `JWT_SECRET`：JWT 密钥，用于用户认证；
- 其余配置项（如 SMTP 配置）存储在数据库中，提供接口进行修改；

### 滚动备份 rsync 双远程

rsync 不支持远程到远程的直接传输。当源和目标都是远程时，服务所在主机作为中继：

- **阶段1**：rsync pull 远程源 → 本地中继缓存目录，带 `--link-dest` 引用本地上次缓存
- **阶段2**：rsync push 本地中继缓存 → 远程目标，带 `--link-dest` 引用远程上次快照
- 缓存目录只保留最新一份，用于下次增量引用
- 两阶段都是增量传输，网络开销 ≈ 单次增量的 2 倍
- 前端标记此场景为中继模式，提示用户确保服务器有足够临时磁盘空间

### 容灾率计算

- 容灾率用于衡量实例当前“可恢复能力”，不是简单的历史成功率；
- 综合评分公式：`容灾率 = 0.35 × 备份新鲜度 + 0.30 × 恢复点可用性 + 0.20 × 冗余与隔离度 + 0.15 × 执行稳定性`；
- 满分 100 分，展示时同时显示分数、等级、主要扣分原因；

#### 备份新鲜度

- 基于当前实例所有启用计划中最严格的备份周期作为基线；
- 取最近一次成功且可恢复的备份时间与当前时间比较；
- 越接近计划周期上限，分数越高；超过多个周期未产生有效备份时，分数快速下降；
- 如果实例没有启用计划、仅依赖手动触发，则该项分数上限降低，并标记“无自动计划”；

#### 恢复点可用性

- 基于当前启用策略是否存在未过期且成功的恢复点；
- 每个启用策略至少应有一个可用恢复点；
- 若至少一个策略拥有多个连续可用恢复点，可额外加分；
- 如果实例未配置启用策略，则该项记为 0 分；

#### 冗余与隔离度

- 评估备份目标是否具备多目标、跨主机、异地等冗余能力；
- 至少两个健康目标且其中一个为远程/异地，得分最高；
- 仅单一本地目标时得分较低；
- 目标不可达、容量不足、健康检查失败时明显扣分；

#### 执行稳定性

- 基于最近一段时间的备份执行成功率、连续失败次数、当前阻塞性风险计算；
- 阻塞性风险包括目标不可达、容量耗尽、凭证错误等；
- 该项权重低于新鲜度和恢复点可用性，避免“历史成功率高”掩盖“当前不可恢复”的问题；

#### 等级定义

- 85-100：安全；
- 70-84：注意；
- 40-69：风险；
- 0-39：危险；

#### 扣分原因展示

- 最近冷备份已超过计划周期；
- 只有单一本地备份目标；
- 备份目标剩余容量不足；
- 最近连续多次执行失败；
